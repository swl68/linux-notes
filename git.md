## Создание первого проекта
### Инициализировать Git-репозиторий в директории проекта: 
```git init```
### Добавить все файлы в индекс, указав "." (подготовка к коммиту): 
```git add .```
### Если нужно добавить только определённые файлы, указать их явно: 
```git add file_name.extension```
### Сделать первый коммит с описательным сообщением: 
```git commit -m "Initial commit: создание проекта"```
### Связать локальный репозиторий с удалённым на Gitea:
```git remote add originSSH git@gitea.com:username/repository_name.git```
### Отправить изменения в удалённый репозиторий, ветка master или main в зависимости от настройки. Флаг -u устанавливает upstream-ветку, связывая локальную ветку с удалённой:
```git push -u origin master```


## Подтянуть изменения с мастера, когда уже написано много кода в своей ветке
### Делаем комит изменений из своей ветки без публикации (push):
```git add .```
и
```git commit -m "Text"```
### Переключаемся на локальную ветку master:
```git checkout master```
### и подтягиваем изменения с удаленного репозитория:
```git pull orignSSH master```
### Переключаемся в нашу, рабочую ветку:
```git checkout test_merge```
### Выполняем для добавления новых данных уже в свою рабочую ветку(разрешить конфликты при возникновении):
```git merge master```
### Далее можем переключится в рабочую ветку и работать в обычном режиме
